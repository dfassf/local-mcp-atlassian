# Jira 티켓 작성 가이드

## 이슈 타입

| 타입 | 용도 |
|------|------|
| Task | 일반 작업 |
| Bug | 버그 수정 |
| Story | 기능 구현 (사용자 관점) |
| Epic | 대규모 기능 묶음 |
| Sub-task | 상위 작업의 하위 항목 |

## 이슈 생성 양식

### 제목 (Summary)
```
[카테고리] 작업 내용 요약
```

예시:
- `[API] 사용자 인증 API 구현`
- `[FIX] 로그인 시 토큰 만료 오류`
- `[REFACTOR] 공통 유틸 함수 분리`

### 설명 (Description)
```
## 목적
- 왜 이 작업이 필요한지

## 작업 내용
- 구체적으로 무엇을 할 것인지

## 참고
- 관련 문서, 링크 등
```

## 우선순위

| 우선순위 | 용도 |
|----------|------|
| Highest | 즉시 처리 필요 (장애, 긴급 버그) |
| High | 금주 내 처리 |
| Medium | 일반 작업 (기본값) |
| Low | 여유 있을 때 |
| Lowest | 백로그 |

## 상태 전환

```
해야 할 일 → 진행 중 → 완료
```

### 상태별 의미
- **해야 할 일**: 아직 시작 안 함
- **진행 중**: 현재 작업 중
- **완료**: 작업 완료 (리뷰/배포 포함)

## 댓글 작성 규칙

### 작업 시작
```
✅ 작업 시작합니다.
```

### 진행 상황 공유
```
📝 현재 상황:
- 완료: A 기능 구현
- 진행 중: B 기능 테스트
- 예정: C 기능 연동
```

### 이슈 발생
```
⚠️ 이슈 발생:
- 문제: [문제 설명]
- 원인: [원인 분석]
- 해결 방안: [제안]
```

### 작업 완료
```
✅ 작업 완료
- PR: [링크]
- 배포: dev/prod
```

## 이슈 연결 (Link)

| 링크 타입 | 의미 | 사용 예시 |
|-----------|------|-----------|
| Blocks | A가 완료되어야 B 진행 가능 | 인프라 → 배포 |
| Relates | 관련 있음 | 같은 기능의 FE/BE |
| Duplicate | 중복 이슈 | 동일 버그 리포트 |

## 라벨

| 라벨 | 용도 |
|------|------|
| frontend | 프론트엔드 작업 |
| backend | 백엔드 작업 |
| urgent | 긴급 |
| blocked | 블로커 있음 |

## MCP 도구 사용 예시

### 이슈 생성
```
jira_create_issue:
  projectKey: "PMFG"
  summary: "[API] 사용자 인증 구현"
  issueType: "Task"
  priority: "Medium"
  assignee: "your.email@company.com"
```

### 상태 변경
```
jira_transition_issue:
  issueKey: "PMFG-123"
  transitionName: "진행 중"
```

### 댓글 추가
```
jira_add_comment:
  issueKey: "PMFG-123"
  body: "✅ 작업 시작합니다."
```
